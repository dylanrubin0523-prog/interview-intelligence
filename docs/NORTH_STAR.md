# North Star

## Mission

Help students navigate recruiting with trustworthy, structured, community-contributed intelligence — replacing scattered, unverifiable forum threads and word-of-mouth with reports and data that clearly show their own reliability.

This mission is durable and not tied to any single phase. Everything below explains how the current, committed work relates to it, and how future ideas are held without being mistaken for commitments.

## Current: Phase 1

Phase 1 (`docs/PRODUCT_SPEC.md`) is the concrete, committed implementation of the first slice of this mission: a normalized company/position directory, structured and moderated interview reports, a private application tracker, and text-based AI prep questions with honest provenance labeling — scoped to business-track recruiting (accounting, finance, consulting, wealth management). The GitHub milestones and issues in this repository are the only authoritative record of what's actually being built and when.

## Future vision: Recruiting Intelligence

Phase 1 is a foundation, not the ceiling. The longer-term vision is a broader **Recruiting Intelligence** platform — see `docs/RECRUITING_INTELLIGENCE.md` for the detailed picture — that layers personalization, timely signal, and recommendations on top of the structured data Phase 1 establishes. Two supporting long-term documents exist alongside it:

- `docs/AI_STRATEGY.md` — how AI capabilities grow beyond Phase 1's single text-generation use case, and the principles that constrain that growth.
- `docs/MONETIZATION.md` — how the product might eventually sustain itself financially, and why that's deliberately deferred.

## Long-term ideas are not commitments

Nothing in `RECRUITING_INTELLIGENCE.md`, `AI_STRATEGY.md`, or `MONETIZATION.md` changes Phase 1 scope, the current GitHub milestones, or `docs/DECISIONS.md`. These documents exist so good ideas aren't lost, not to schedule them. An idea graduates from "future vision" to "active work" only when it gets its own product spec, its own ADR if it's architectural, and its own GitHub issues — the same process every Phase 1 feature went through. Until then, treat everything in those three documents as explicitly **not** part of the implementation roadmap.
